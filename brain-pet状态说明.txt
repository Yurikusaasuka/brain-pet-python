【脑区颜色对应关系】
每个脑区在大多数状态下保持固定的色调，方便识别。
颜色完全由代码着色流水线控制，PNG 文件仅提供形状蒙版（已中性化为白色）：

  前额叶 (pfc_left / pfc_right)   → 蓝色     #3498DB
  顶叶   (parietal)               → 青绿色   #17A589
  颞叶   (temporal)               → 琥珀金   #D68910
  枕叶   (occipital)              → 深橙色   #CA6F1E
  小脑   (cerebellum)             → 绿色     #27AE60
  脑干   (brainstem)              → 深红色   #922B21   ← 正式脑区，负责基本生命维持
  边缘叶 (limbic)                 → 深红色   #C0392B   ← 合成绘制（无 PNG）
  布洛卡区 (broca)                → 薄荷绿   #1ABC9C   ← 合成绘制（无 PNG）

特殊状态（心流/过载）会打破上述色彩规律以产生强烈视觉信号。

【颜色设计原则】
- 任何状态下绝对不使用灰色表示"不活跃" — 低活跃度用「低强度+自然色」表示
- region PNG 文件在加载时被转换为纯白形状蒙版；base/frame 转换为灰度轮廓
- apply_glow 管道：亮度随强度缩放，色相始终保留（不向灰色偏移）
- 低活跃区域：intensity 0.05–0.20，颜色保持自然色调

【合成绘制区域（无 PNG 文件）】
  布洛卡区：椭圆形，位于额叶右侧（x: 66–80%, y: 40–60%），高斯模糊软化
  边缘叶：C 形弧，位于脑内侧（左侧弧形），内圆掏空，高斯模糊软化


================================================================================
  一、窗口监控与识别机制（2026-06-12 重构）
================================================================================

【两层轮询架构】
  快速层（每 1 秒）：
    - 通过 GetForegroundWindow() 即时获取当前前台窗口句柄（HWND）
    - 延迟从原来最高 2 秒降低至最高 1 秒
    - 使用同一 HWND 传入 GetClassName() 进行游戏窗口类检测

  慢速层（每 3 秒，即每隔 2 次快速轮询触发一次）：
    - 调用 EnumWindows() 枚举所有可见用户窗口（Z 序）
    - 更新「前三窗口」缓存供状态机使用
    - 同时更新系统可见窗口总数（用于过载检测）
    - 前台窗口始终强制排在枚举结果第一位，确保不会被错过

【前三窗口感知模型（Top-3 Window Awareness）】
  - 检测系统中前 3 个可见用户窗口（Z 序）的应用类别
  - 前台窗口（GetForegroundWindow）在枚举时强制插入第一位
  - 工作状态在工作窗口进入前三即激活，不因切换邮件/浏览器而中断
  - 娱乐窗口进入前三时激活娱乐状态
  - 工作 + 娱乐同时在前三 → 混合模式（MIXED）
  - 空闲仅在系统闲置时间超过阈值时触发

【工作连续时长（work_continuous_seconds）】
  - 独立于前台窗口类别变化追踪工作时长
  - 只要工作类应用在前三窗口中存在，计时器持续累加
  - 仅在工作窗口完全离开前三时重置
  - DEEP_FOCUS、FLOW_STATE、FOCUS_STREAK 均使用此计时器
  - 效果：在工作软件之间切换（VS Code → Chrome/GitHub → Terminal）不会打断专注状态

【应用分类识别（三层）】
  第一层：进程名匹配（APP_CATEGORIES / IDE_PROCESSES / GAME_PROCESSES）
  第二层：窗口标题关键词匹配（AI_TITLE_KEYWORDS / WORK_TITLE_KEYWORDS / GAME_TITLE_KEYWORDS）
  第三层：窗口类名匹配（GAME_WINDOW_CLASSES，通过 GetClassName(HWND) 获取）

【工作类标题关键词（WORK_TITLE_KEYWORDS）】
  浏览器标签包含以下内容时识别为 WORK：
    代码托管：github.com、gitlab.com、bitbucket.org
    项目管理：jira、linear.app、asana.com、trello.com、clickup.com、notion.so
    文档协作：confluence、docs.google.com、sheets.google.com、slides.google.com
    设计：figma.com、zeplin.io
    CI/CD：jenkins、circleci.com、travis-ci、github actions、vercel.com
    本地开发：localhost、127.0.0.1
    参考资料：stackoverflow.com、developer.mozilla.org

【游戏识别（三重检测）】
  进程名：Steam、Epic、Origin、Battlenet、Riot 等启动器及主流游戏可执行文件
           （包含：Valorant、CSGO/CS2、Dota2、Overwatch、Fortnite、Apex、PUBG 等）
  标题关键词：steam、游戏、game、gaming、powered by steam
  窗口类名：SDL_app、UnrealWindow、GLFW30、GameOverlayUIApp 等

【AI 助手识别】
  触发条件：以下进程名或窗口标题匹配（浏览器标签亦可触发）：
    进程名：claude, chatgpt, cursor
    标题关键词：claude.ai, chatgpt.com, chat.openai, gemini, copilot, perplexity,
               poe.com, character.ai, you.com
  效果：归入 WORK 类别，状态为 WRITING；状态栏显示「写作中 · AI协作 · 时段」


================================================================================
  二、工作状态（自动检测）
================================================================================

【普通工作  NORMAL_WORK】
  触发条件：工作窗口在前三，work_continuous_seconds 不足 focus_threshold 分钟
  前额叶：蓝色 #3498DB，强度 0.7
  布洛卡区：薄荷绿 #1ABC9C，强度 0.5（合成绘制）
  动画：静止（static）
  视觉增强：饱和度 ×1.1

【深度专注  DEEP_FOCUS】
  触发条件：工作窗口在前三，work_continuous_seconds ≥ focus_threshold（默认 20 分钟）
            工作软件间相互切换不重置计时器
  前额叶：蓝色 #3498DB，强度 1.0
  顶叶：  青绿色 #17A589，强度 0.4
  动画：呼吸（breathe），周期 3000ms（0.3~1.0，额叶始终保持可见，永不完全熄灭）
  视觉增强：亮度 ×1.1，饱和度 ×1.3
  子区域效果：额叶前部（11区，图像左侧偏下）更亮，后部（1-2区，图像右侧）渐暗

【代码调试  CODE_DEBUG】
  触发条件：IDE 在前三 + 窗口标题含错误关键词
  前额叶：深蓝色 #2980B9，强度 0.9
  顶叶：  橙色 #E67E22，强度 0.8
  动画：闪烁（flash），周期 500ms
    - 亮相 40%（t < 0.4），全灭 60%（强度 0.0），对比极强
  视觉增强：亮度 ×1.05，饱和度 ×1.35

【写作 / AI协作  WRITING】
  触发条件：Word、Notion、Obsidian、AI 助手等文字 / 对话软件激活
  布洛卡区：薄荷绿 #1ABC9C，强度 1.0（合成绘制）
  前额叶：  蓝色 #3498DB，强度 0.6
  动画：波浪（wave），周期 3000ms
  视觉增强：饱和度 ×1.15

【开会  MEETING】
  触发条件：Zoom、Teams、Discord、微信、飞书、Telegram、钉钉等通讯软件激活
  颞叶：琥珀金 #D68910，强度 1.0
  布洛卡区：薄荷绿 #1ABC9C，强度 0.8（合成绘制）
  动画：脉冲（pulse），周期 1500ms

【过载  OVERLOAD】  ← 最高优先级
  触发条件：可见窗口数 ≥ 10（慢速层每 3 秒更新），或 3 分钟内切换应用 ≥ 15 次
  所有脑区：红色 #E74C3C，强度 1.0（全脑漫红）
  动画：不规则抖动（irregular），周期 200ms
    - 范围 0.0~1.0（原 0.3~1.0），抖动更剧烈
  视觉增强：亮度 ×1.2，饱和度 ×1.2
  气泡提示：「事情有点多… 😵」（每次进入提示一次）

【混合模式  MIXED】  ← 优先级 8
  触发条件：前三窗口中同时有工作类和娱乐类应用
  前额叶：蓝色 #3498DB，强度 0.7
  枕叶：  深橙色 #CA6F1E，强度 0.5
  边缘叶：深红色 #C0392B，强度 0.4（合成绘制）
  动画：波浪（wave），周期 2500ms
  说明：不计入专注时长和番茄钟，代表工作/娱乐混合状态

【游戏中  GAMING】  ← 优先级 8
  触发条件（三选一）：
    ① 游戏进程名匹配（Steam、Epic、Riot 等 30+ 款主流游戏）
    ② 窗口标题含 game / steam / 游戏 等关键词
    ③ 窗口类名匹配（SDL_app、UnrealWindow、GLFW30 等）
  小脑：绿色 #27AE60，强度 0.9
  枕叶：深橙色 #CA6F1E，强度 0.8
  顶叶：青绿色 #17A589，强度 0.7
  前额叶：蓝色 #3498DB，强度 0.5
  动画：脉冲（pulse），周期 800ms
  自动隐藏：可在设置中开启「打开游戏时自动隐藏桌宠」

【心流  FLOW_STATE】  ← 第二优先级
  触发条件：工作类应用在前三窗口连续 45 分钟以上（需开启「心流提醒」）
            使用 work_continuous_seconds 计时：工作软件间切换不重置
  所有脑区：金色 #FFD700，强度 1.0
  动画：随机闪烁（sparkle），周期 400ms
  视觉增强：亮度 ×1.3，饱和度 ×1.6（最高增强）
  气泡提示：「心流状态！🔥」

【专注连胜  FOCUS_STREAK】  ← 优先级 2.5
  触发条件：完成 ≥ N 个番茄钟（N 可配置，默认 3），
            且 work_continuous_seconds ≥ focus_threshold
  前额叶（左右）：金色 #FFD700，强度 1.0
  顶叶：琥珀色 #F39C12，强度 0.6
  动画：随机闪烁（sparkle），周期 400ms
  视觉特效：脑图上方显示 3 个金色小三角形皇冠（▲▲▲）

【拖延中  PROCRASTINATION】
  触发条件：工作软件与娱乐软件交替切换 ≥ 10 次（30 分钟窗口内）
  前额叶：蓝色 #3498DB，强度 0.7
  边缘叶：深红色 #C0392B，强度 0.7（合成绘制）
  动画：随机抖动（jitter），周期 300ms
    - 范围 0.0~1.0，抖动幅度最大

【压力过高  STRESS_HIGH】
  触发条件：深夜 0–6 点仍在使用工作软件
  边缘叶：红色 #E74C3C，强度 1.0（合成绘制）
  前额叶：红色 #E74C3C，强度 0.6
  脑干：  深红色 #922B21，强度 0.4（应激反应，战逃模式）
  动画：快速闪烁（flash），周期 300ms
    - 亮相 40%（t < 0.4），全灭 60%（强度 0.0）
  气泡提示：「休息一下？💙」

【多巴胺耗尽  DOPAMINE_CRASH】
  触发条件：SOCIAL_MEDIA 持续 ≥ 60 分钟
  边缘叶：极弱深红 #C0392B，强度 0.08（合成绘制）
  前额叶：极弱灰蓝 #85B0D0，强度 0.05
  过渡动画：边缘叶从强度 0.9 线性淡出至 0.08，历时 3 秒（90帧）
  气泡提示：「多巴胺有点耗尽了… 出去走走？」
  视觉增强：亮度 ×0.65，饱和度 ×0.45（高度脱色但保留色相）

【未知活动  UNRECOGNIZED_ACTIVE】  ← 优先级 8（最低识别状态）
  触发条件：用户活跃（闲置时间 < 10 秒），但前三窗口均无法识别类别
  前额叶（左右）：浅蓝色 #85C1E9，强度 0.25
  动画：呼吸（breathe），周期 3000ms

================================================================================
  八、动画类型说明（animation.py）
================================================================================

  pulse     - 正弦波，0.0~1.0 循环（底部完全熄灭，对比强烈）
  breathe   - 平方正弦，0.3~1.0（底部保留 30% 亮度，模拟呼吸节奏）
  flash     - 硬切：亮相 40%（t<0.4，强度 1.0），全灭 60%（强度 0.0）
              对比极强，用于 CODE_DEBUG、VIDEO_HORROR、STRESS_HIGH
  wave      - 正弦波，0.4~1.0（比 pulse 底部更亮）
  static    - 固定 1.0（无动画）
  irregular - 随机 0.0~1.0 每帧（不规则抖动，用于 OVERLOAD）
  sparkle   - 随机选 [1.0, 1.0, 0.9, 0.0]（主要全亮，偶尔全灭产生闪烁感）
  jitter    - 随机 0.0~1.0 每帧（同 irregular，用于 PROCRASTINATION）


================================================================================
  九、时段划分标准（How time periods are defined）
================================================================================

时段划分规则（以本地 datetime.now().hour 实时判断，每 1 秒更新）：

  清晨     06:00 – 09:00   亮度 78%，预热阶段（前 30 分钟从 20% 线性渐亮）
  上午高峰 09:00 – 12:00   亮度 100%，最佳状态
  午后困倦 12:00 – 14:00   亮度 88%，额外叠加极慢琥珀色微脉冲（drowsy 效果）
  下午恢复 14:00 – 17:00   亮度 95%
  傍晚     17:00 – 19:00   亮度 82%
  晚间     19:00 – 22:00   亮度 85%
  夜间     22:00 – 00:00   亮度 72%，首次进入提示「快深夜了… 🌙」
  深夜     00:00 – 06:00   亮度 62%，首次进入提示「还没睡？😴」

判定规则：
  - 以本地系统时间（datetime.now().hour）为准
  - 时段在每次状态刷新时（每 ~1 秒）实时判断，不缓存
  - 同一运行周期内每个时段的气泡提示仅触发一次
    （用 set() 记录已提示过的时段，程序重启后重置）
  - 过载（OVERLOAD）和心流（FLOW_STATE）状态下跳过亮度叠加
  - 午后困倦微脉冲也跳过 OVERLOAD 和 FLOW_STATE


================================================================================
  十、番茄钟系统（主动阶段机器）
================================================================================

【设计思路】
  番茄钟采用「主动确认」模式：用户通过点击按钮驱动每个阶段的切换，
  而不是由程序被动检测用户是否离开了电脑。

【五个阶段（_pom_phase）】

  ''（初始 / 重置）
    进入：程序启动，或被打断性娱乐行为重置
    退出：进入工作状态时自动推进至 'WORK'

  'WORK'（工作计时中）
    进入：用户开始使用工作类应用
    状态栏显示：🍅 MM:SS（实时倒计时，按已用时间显示剩余量）
    退出条件 A：工作时长到达 N 分钟 → 推进至 'WORK_DONE'
    退出条件 B：出现「打断状态」(见下) → 重置至 ''

  'WORK_DONE'（工作完成，等待用户确认休息）
    状态栏显示：🛁 开始休息（可点击按钮）
    用户点击后 → 推进至 'REST'
    打断状态出现 → 重置至 ''

  'REST'（休息倒计时中）
    状态栏显示：😴 MM:SS（休息剩余时间）
    休息时长到达 M 分钟 → 推进至 'REST_DONE'

  'REST_DONE'（休息结束，等待用户确认继续工作）
    状态栏显示：💪 重新开始工作（可点击按钮）
    用户点击（或进入工作状态）→ 完成一个番茄钟，推进至 'WORK'
    打断状态出现 → 重置至 ''

【计入工作时间的状态（_WORK_STATES）】
  在以下任意状态中切换均不重置番茄钟计时器：
    NORMAL_WORK、DEEP_FOCUS、CODE_DEBUG、WRITING、
    MEETING、MUSIC_WORK
  说明：开会（MEETING）和边听音乐边工作（MUSIC_WORK）均属于工作范畴，
        在这些状态间切换视为工作连续进行。

【打断性状态（_DISTRACT_STATES，会重置番茄钟至 ''）】
    GAMING、VIDEO_UNKNOWN、VIDEO_DOCUMENTARY、VIDEO_ENTERTAINMENT、
    VIDEO_HORROR、SOCIAL_MEDIA、DOPAMINE_CRASH、MIXED、PROCRASTINATION

【暂停性状态（既非工作也非打断，番茄钟暂停而不重置）】
    SHORT_REST、DEEP_AWAY、IDLE、UNRECOGNIZED_ACTIVE 等
  效果：短暂离开电脑（小憩、上厕所）不会重置番茄钟，工作时间从暂停处继续。

【番茄钟完成（一个完整周期）】
  'WORK' → 'WORK_DONE'（用户确认）→ 'REST' → 'REST_DONE'（用户确认）→ 'WORK'
  完成时：_pom_streak +1，daily_stats.pomodoro_count +1
  气泡提示：「🍅 番茄钟完成！专注力+1」

【可配置参数（右键 → 🍅 番茄钟设置）】
  工作时长 N：25 分钟（范围 10–60）
  短休息时长 M：5 分钟（范围 1–15）
  连胜触发数：3 个（范围 2–10）

配置存储路径：settings.json → "pomodoro" 键
{
  "work_minutes": 25,
  "break_minutes": 5,
  "streak_threshold": 3
}

完成 ≥ 连胜触发数个番茄钟后，深度专注状态自动升级为「专注连胜」。


================================================================================
  十一、今日大脑报告
================================================================================

入口：右键菜单 → 📊 今日大脑报告

内容：
  - 最活跃脑区排行（横向彩色进度条，按各脑区今日总激活量排序）
  - 今日专注总时长、最长连续专注时长
  - 番茄钟完成数、应用切换次数
  - 今日状态分布横条（各状态按时长着色）
  - 今日首次激活时间：HH:MM（替代原睡眠追踪数据）

数据每 30 秒自动累计一次，每天午夜自动归档（保留最近 7 天）。
统计存储在 settings.json 的 "daily_stats" 和 "history" 键下。

注意：睡眠追踪与恢复模式功能已移除（版本 2026-06-08）。
      「SLEEPING」状态本身保留（手动 + 自动触发），但不再计算睡眠债务。


================================================================================
  十二、自动隐藏行为
================================================================================

可在「⚙ 设置」中开启以下自动隐藏选项（默认均关闭）：

  打开游戏时自动隐藏桌宠：
    - GAMING 状态下自动 withdraw 主窗口
    - 系统托盘图标保持可见，点击 Show / Hide 恢复

  全屏视频时自动隐藏桌宠：
    - 当前台窗口占满全屏且状态为视频类时自动隐藏
    - 退出全屏或状态变化后自动恢复显示

手动操作：
  右键菜单 → 🔽 隐藏到托盘：立即隐藏到系统托盘
  托盘图标右键 → Show / Hide：切换显示/隐藏


================================================================================
  十三、状态优先级（从高到低）
================================================================================

  1.   过载（OVERLOAD）                      ← 最高，覆盖一切
  2.   心流（FLOW_STATE）                    ← work_continuous_seconds ≥ 45 分钟
  2.5. 专注连胜（FOCUS_STREAK）              ← ≥N 个番茄钟后激活（N 可配置）
  3.   手动状态（运动 / 用餐 / 睡眠 / 散步 / 创意）
  4.   拖延（PROCRASTINATION）
  5.   压力过高（STRESS_HIGH）
  6.   长时间离开（DEEP_AWAY）
  7.   小憩（SHORT_REST）
  8.   工作 / 娱乐 / 混合类状态（按前三窗口内容分类）：
       8a. 工作状态（前三有工作，无娱乐）
           NORMAL_WORK / DEEP_FOCUS / CODE_DEBUG / WRITING / MEETING
           DEEP_FOCUS 使用 work_continuous_seconds（工作软件切换不打断）
       8b. MIXED（工作 + 娱乐同时在前三）
       8c. GAMING（三重检测：进程名 + 标题关键词 + 窗口类名）
       8d. 娱乐状态（VIDEO_* / MUSIC_WORK / SOCIAL_MEDIA）
           DOPAMINE_CRASH 覆盖 SOCIAL_MEDIA（持续 ≥60 分钟）
       8e. UNRECOGNIZED_ACTIVE（活跃但无识别类别）
  9.   空闲（IDLE）                          ← 兜底默认

  叠加层（在上述状态基础上额外修改显示，不替换状态）：
    - 时间叠加（亮度调节，不影响 OVERLOAD/FLOW_STATE）
    - 午后困意微脉冲（12:00-14:00，不影响 OVERLOAD/FLOW_STATE）


================================================================================
  十四、子区域效果（空间梯度发光）
================================================================================

  深度专注时：额叶前部（11区，图像左侧偏下）更亮，后部（1-2区，图像右侧）渐暗
  创意模式时：右额叶前部有流动光斑，模拟 11 区的随机神经激活
  散步模式时：全脑各区各自以独立相位漫游（wander），模拟 DMN 随机激活

  颞叶子区域：
  听音乐时：颞叶偏上部（10区方向）更亮，模拟听觉皮层激活
  休息时：  颞叶偏中下（8区方向）更亮，模拟海马区的默认活动


================================================================================
  十五、可配置参数（右键 → ⚙ 设置）
================================================================================

  深度专注阈值（分钟）：默认 20，可调 5-60
  休息检测阈值（分钟）：默认 5，可调 1-30
  过载窗口数阈值：       默认 10，可调 3-30（窗口数由慢速层每 3 秒更新一次）
  工作软件列表：         可手动添加/删除
  视频类型询问：         开/关（默认开）
  心流提醒：             开/关（默认开）
  深夜提醒：             开/关（默认开）
  显示气泡：             开/关（默认开）
  启用动画：             开/关（默认开）
  打开游戏时自动隐藏：   开/关（默认关）
  全屏视频时自动隐藏：   开/关（默认关）
  大脑尺寸：             S / M / L

  番茄钟设置（右键 → 🍅 番茄钟设置）：
  工作时长：默认 25 分钟，可调 10-60
  短休息时长：默认 5 分钟，可调 1-15
  连胜触发数：默认 3 个，可调 2-10

  注意：所有阈值类滑块均为整数步进（resolution=1），不会出现小数值。
